Project Goals and Context

The primary goal of OIF is to provide an open-source, standards-based solution that allows new L2 chains to implement intent-based bridging without vendor lock-in. The aim is to make adoption as easy as possible for new chains to deploy and use the system.

Key Priorities

  1. Easy onboarding for new L2 chains;
  1. Open standards, no vendor lock-in;
  1. Extensible architecture for future order types and chain technologies.

Beta Requirements

Goals

  • Functional solver that can process basic cross-chain intents
  • Support for ERC-7683 standard orders (primary focus)
  • Basic monitoring and operational tooling
  • Foundation for production blockchain integration

Core Requirements

Order Type Support

  • Primary: ERC-7683 standard orders
  • Secondary: Consider Li.fi standard interface only if beneficial to ecosystem (not vendor-specific)
  • Future: Additional OIF order types (Bridge Order, Limit Order) as defined by smart contract team

Settlement Architecture

  • Support for Arbitrum cross-chain broadcaster as committed settlement mechanism
  • Two-part settlement flow:
    1. Generate proof on destination chain that solver fulfilled intent
    1. Send proof back to source chain to release funds from
  • Support for two mechanisms:
    • Compact resource locks
    • Simpler generic escrow

Key Architectural Decisions

Delivery Strategy

  • Preferred: Use third-party services (Alchemy) or OSS relayers rather than building custom solutions
  • Avoid: Custom nonce management, gas optimization, transaction tracking
  • Rationale: Focus on reliability and "run and forget" operations

Order Distribution

  • Phase 1: Simple "first solver to execute gets paid" model
  • Phase 2: Solver reservation system (see reservation system details below)
  • Not committed: Auction-based distribution for this phase

Key Management

  • Secure storage for private keys and API keys
  • Assume trusted solver environments
  • Consider third-party key management for production

Beta Success Criteria

  • Can listen for ERC-7683 intents on EVM chains
  • Can validate and fulfill profitable intents
  • Can claim settlements using Arbitrum cross-chain broadcaster
  • Basic monitoring and debugging capabilities
  • Easy deployment process for new L2 chains

MVP Requirements

Goals

  • Production-ready solver with comprehensive order type support
  • Advanced validation and optimization capabilities
  • Complete onboarding solution for new L2 chains
  • Real users can start using the system

Enhanced Requirements (Building on Beta)

Order Type Expansion

  1. Complete OIF Order Type Support - All OIF-defined order types
  1. Order Type Router - Unified handling system
  1. Modular Order Processing - Plugin architecture for future types

Solver Reservation System

  • Order contract supports RESERVER role
  • RESERVER can assign orders to specific solvers with expiration blocks
  • Only assigned solver can complete order during reservation period
  • Simple off-chain reservation service run by L2 admins
  • Naive round-robin distribution among registered solvers initially

Advanced Features

  1. Validation Plugin System - Extensible validators
  1. WebSocket Support - Real-time order updates
  1. Performance Optimization - Throughput improvements
  1. Multi-chain EVM Support - Comprehensive chain coverage

Liquidity and Fallback Strategy (not required for MVP but architectural consideration)

  • Problem: What happens when no solver has liquidity?
    • Destination chain lacks liquidity
    • Solvers don't have required token liquidity
    • New/uncommon tokens
  • Approach:
    • Modular fallback system: Plugin-based or strategy-based approach
    • Multi-hop solutions: Swap → Bridge → Swap on destination
    • Third-party chain routing: Route through high-liquidity chains
    • Configurable strategies: Allow solvers to set risk tolerance and fallback preferences

Production Excellence

  1. Easy Deployment Tools:
    • CLI for contract deployment
    • CLI for solver deployment and management
    • "Push-button" experience for new L2s
  1. Documentation:
    • Complete operator guides
    • Integration documentation

MVP Success Criteria

  • Supports all OIF order types on EVM chains
  • Solver reservation system operational
  • Complete onboarding solution for new L2 chains
  • Real users successfully processing intents
  • Production-grade reliability and monitoring
  • Successfully processing intents on multiple EVM chains

Future Roadmap (Post-MVP)

Q4 2024 - Advanced Features

  • Liquidity Management: Automated rebalancing systems and fallback strategies
  • Solver Auction System: Competitive bidding for orders
  • Advanced Monitoring: Dashboards and alerting systems
  • Network Expansion: Consider non-evm abstractions

Q1 2025 - Scale & Optimize

  • Capital Efficiency: Bonded solver model (Across Prime approach)
  • Performance Optimization: High throughput batching
  • Horizontal Scaling: Multi-solver coordination
  • Advanced Analytics: Comprehensive reporting and insights
